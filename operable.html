2. Operable - Segundo Principio

Los componentes de la interfaz de usuario y la navegación deben ser operables, la interfaz no puede exigir una interacción al usuario que no puede realizar.


2.1 Accesible por teclado

Proporcionar acceso a toda la funcionalidad mediante el teclado.


2.2 Tiempo suficiente

Proporcionar a los usuarios el tiempo suficiente para leer y usar el contenido. Normalmente, un usuario con discapacidad tardará más tiempo que la mayoría de usuarios para realizar una tarea: pueden necesitar más tiempo para la lectura, puede que deban ajustarse a la velocidad de un software que usen, etc.


2.3 Convulsiones

No diseñar contenido de un modo que se sepa podría provocar ataques, espasmos o convulsiones.


2.4 Navegable

Proporcionar medios para ayudar a los usuarios a navegar, encontrar contenido y determinar dónde se encuentran.
